## Learnings

### Technical Insights

1. **Data Structure:** Decision logs use `generated_at` not `timestamp`
2. **Price Data:** Available in `evidence.sizing.current_price`
3. **Fallback:** Can use `evidence.latest_close` when current_price missing
4. **Filtering:** 508/647 decisions meet confidence >= 0.70 threshold

### Process Improvements

1. Early data validation saves debugging time
2. Incremental testing (5 separate test suites) ensures quality
3. Modular architecture enables parallel development
4. Git commits at logical milestones provide good rollback points

### Performance Considerations

1. File-based price cache needed for historical data
2. Memory cache improves repeated access
3. Daily simulation loop scales well with data size
4. Domain constraints reduce search space effectively
