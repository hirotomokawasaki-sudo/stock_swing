# コンソール チャート詳細ガイド

**最終更新**: 2026-04-26

---

## 📊 **概要**

Stock Swing Console には **4つの主要チャート** があります：

1. **Equity (資産推移)**
2. **Drawdown % (ドローダウン率)**
3. **Open Positions (オープンポジション数)**
4. **Signals/Orders (シグナル vs 注文数)**

各チャートは **過去30日分のデータ** を表示します。

---

## 1️⃣ **Equity Chart (資産推移)**

### **何を表示するか**
- 日次の **総資産額 (Equity)** の推移
- Broker account の equity 値を時系列で表示

### **データ構造**
```json
{
  "ts": "2026-04-26T00:00:00",  // タイムスタンプ (日付)
  "value": 105533.91             // Equity (USD)
}
```

### **データソース**
- `trading.daily_snapshots[].equity`
- 毎日の snapshot から取得
- Broker API の account equity

### **読み方**
- **上昇トレンド**: 利益が出ている、運用が順調
- **下降トレンド**: 損失が出ている、見直しが必要
- **横ばい**: 利益も損失もない状態

### **実例 (2026-04-26)**
```
2026-04-17: $100,037.26
2026-04-18: $101,756.02
2026-04-19: $101,321.46
...
2026-04-26: $105,533.91
```

**分析**: 約5%の上昇 ($100k → $105k)

## 2️⃣ **Drawdown % Chart (ドローダウン率)**

### **何を表示するか**
- **ドローダウン率**: 過去最高値 (Peak Equity) からの下落率
- リスク管理の重要指標

### **計算式**
```python
drawdown_pct = (peak_equity - current_equity) / peak_equity

例:
peak_equity = $110,000
current_equity = $105,533
drawdown_pct = (110,000 - 105,533) / 110,000 = 4.06%
```

### **データ構造**
```json
{
  "ts": "2026-04-26T00:00:00",  // タイムスタンプ
  "value": 0.0406                // Drawdown率 (0.0406 = 4.06%)
}
```

### **データソース**
- `trading.summary.peak_equity`: 過去最高値
- `trading.daily_snapshots[].equity`: 日次 equity
- 計算: `(peak - current) / peak`

### **読み方**
- **0%**: 過去最高値を更新中 (理想的)
- **0-5%**: 正常範囲 (許容範囲内)
- **5-10%**: 要注意 (監視強化)
- **10%+**: 危険 (対策必要)

### **重要性**
- リスク管理の最重要指標
- 最大 drawdown (Max DD) を監視
- 10% を超えたら戦略見直し

### **実例 (過去のMax DD)**
```
Max Drawdown: -8.17% (記録)
Current: ~4.06% (推定)
```

## 3️⃣ **Open Positions Chart (オープンポジション数)**

### **何を表示するか**
- **オープンポジション数**: 保有している銘柄の数
- ポートフォリオの分散度を示す

### **データ構造**
```json
{
  "ts": "2026-04-26T00:00:00",  // タイムスタンプ
  "value": 10                    // ポジション数
}
```

### **データソース**
- `positions.count`: 現在のポジション数
- 毎日の snapshot で記録

### **読み方**
- **0-5銘柄**: 集中投資 (リスク高、リターン高)
- **5-10銘柄**: バランス型 (適度な分散)
- **10-20銘柄**: 分散投資 (リスク低、管理複雑)
- **20銘柄+**: 過度な分散 (リターン希薄化)

### **現在の状況 (2026-04-26)**
```
Open Positions: 10 銘柄
- PLTR (10.0%), DDOG (8.8%), PANW (8.3%), ...
```

**分析**: 適度な分散、最大 10% 制限で集中リスクを管理

## 4️⃣ **Signals/Orders Chart (シグナル vs 注文数)**

### **何を表示するか**
- **Signals**: 戦略が生成したシグナル数 (BUY/SELL 候補)
- **Orders**: 実際に実行された注文数
- **Conversion Rate**: `Orders / Signals` の比率

### **データ構造**
```json
{
  "ts": "2026-04-26T00:00:00",  // タイムスタンプ
  "signals": 17,                 // シグナル数
  "orders": 1                    // 注文数
}
```

### **データソース**
- `signals`: `daily_snapshots[].signals_generated`
- `orders`: `daily_snapshots[].orders_submitted`

### **読み方**

#### **Signals (シグナル数)**
- 戦略が「これは買い/売りのチャンス」と判断した回数
- 多い = 市場に機会が多い
- 少ない = 市場が静か、または戦略が保守的

#### **Orders (注文数)**
- 実際に broker に注文を出した回数
- Signals のうち、リスク検証を通過したもの

#### **Conversion Rate (実行率)**
```python
conversion_rate = orders / signals

例:
signals = 17
orders = 1
conversion_rate = 1/17 = 5.9%
```

### **Conversion Rate の意味**

| Conversion Rate | 意味 | 評価 |
|-----------------|------|------|
| **0-10%** | ほとんど拒否される | ⚠️ リスクゲートが厳しすぎる |
| **10-20%** | 慎重に選別 | ⚠️ やや保守的 |
| **20-30%** | バランス良好 | ✅ 理想的 |
| **30-50%** | 積極的に実行 | ⚠️ リスク管理確認 |
| **50%+** | ほぼ全実行 | ❌ リスクゲート機能していない |

### **現在の状況 (2026-04-26)**
```
Today's execution:
  Decisions: 14
  Submissions: 4
  Conversion Rate: 28.6% ✅ (理想的)

Cumulative (過去):
  Decisions: 500
  Submissions: 66
  Conversion Rate: 13.2% ⚠️ (低い)
```

**分析**: 
- 昨日の改善後、28.6% に向上 (目標達成)
- 累積では 13.2% (過去のデータ含む)

## 📈 **チャートの活用方法**

### **1. 日常モニタリング (毎朝5分)**

```bash
# 4つのチェックポイント
1. Equity: 上昇トレンドか？
2. Drawdown: 5% 以下か？
3. Positions: 5-15銘柄の範囲内か？
4. Conversion: 20-30% の範囲内か？
```

### **2. 異常検知**

#### **異常パターン 1: Equity急落**
- **症状**: Equity が1日で5%以上下落
- **原因**: 大きな損失取引、市場暴落
- **対策**: ポジション確認、Stop loss 確認

#### **異常パターン 2: Drawdown 10%超**
- **症状**: Drawdown が 10% を超える
- **原因**: 連続損失、戦略不調
- **対策**: 戦略見直し、パラメータ調整

#### **異常パターン 3: Positions 0**
- **症状**: ポジション数が 0 になった
- **原因**: 全ポジション決済、市場撤退
- **対策**: 原因確認 (意図的 or バグ)

#### **異常パターン 4: Conversion 0%**
- **症状**: Signals は出るが Orders が 0
- **原因**: リスクゲートが全拒否
- **対策**: パラメータ確認 (position size, sector limit など)

### **3. パフォーマンス評価 (週次)**

```python
# 週次チェックリスト
1. Equity: 週間 +/- % を計算
2. Drawdown: 週間 Max Drawdown 確認
3. Positions: 平均保有銘柄数
4. Conversion: 週間平均 Conversion Rate
```

## 📊 **チャートの制限事項**

### **1. データ保持期間**
- **現在**: 過去30日分のみ
- **理由**: `daily_snapshots[-30:]` で制限
- **改善案**: 全期間データを保存、UI で期間選択

### **2. Open Positions の固定値**
- **問題**: 現在の実装では全日同じ値 (`positions.count`)
- **正しくは**: 各日の実際のポジション数
- **原因**: Daily snapshot に positions count が保存されていない
- **改善案**: Snapshot に positions count を追加

### **3. Drawdown の計算**
- **現在**: 累積 peak_equity からの下落率
- **改善案**: 期間別 drawdown (日次, 週次, 月次)
