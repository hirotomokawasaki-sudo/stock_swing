## Key Design Patterns

### Safety Mechanisms
- **Raw Immutability**: Default fail on overwrite
- **Fail Closed**: Runtime mode, invalid stages → explicit errors
- **Paper-Safe Broker**: Live mode blocked at initialization
- **No Auto-Apply**: All parameter changes require operator approval
- **Kill Switch**: Emergency stop with fail-safe defaults
- **Audit Trail**: Comprehensive logging of all operations
